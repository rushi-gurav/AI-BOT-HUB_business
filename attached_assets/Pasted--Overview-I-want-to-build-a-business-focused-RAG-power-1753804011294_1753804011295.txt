🧩 Overview
I want to build a business-focused, RAG-powered AI chatbot builder using:
•	Frontend: React (with JSX/TSX)
•	Backend: Node.js + Express
•	Database: PostgreSQL
🚀 The entire system must be deployable and fully functional inside Replit’s free plan, with appropriate optimizations or constraints as needed. This is intended to serve as a foundation for a real startup MVP — so practical functionality, clean architecture, and working integrations are more important than visual polish or extras like voice input.
________________________________________
🎯 Core Objective
Create a system where users can build custom AI bots that:
•	Use RAG logic to answer ONLY from uploaded documents (PDF/DOCX)
•	Allow users to plug in their own API keys (OpenAI, OpenRouter, Gemini, Grok, or Custom model)
•	Embed bots into other websites
•	Install bots to desktop or mobile devices using PWA (Progressive Web App)
🟨 Note to Replit: The PWA installation + embedding bots into external sites are absolutely critical to this platform and must be implemented correctly, even if it adds complexity.

3. 🔧 Backend API (Node.js + Express)
•	POST /api/bots → Create bot (handle file uploads via Multer)
•	GET /api/bots → List user’s bots (anonymous session ID)
•	GET /api/bots/:id → Get bot config
•	POST /api/chat/:botId → Handle message:
o	Use LangChain or custom embedding logic to match context from user’s files
o	Inject matched content into the user’s prompt
o	Send to LLM provider using the user’s API key
o	Return structured response
________________________________________
4. 🧠 Chatbot RAG Logic (Core Requirement)
•	When a user sends a message:
o	System extracts relevant parts of the uploaded docs using vector similarity (via OpenAI embeddings or custom)
o	Injects the matched text into the prompt context
o	Sends the updated prompt to the selected LLM
o	Returns a response based solely on document content
•	Works across all providers (OpenAI, OpenRouter, etc.)
